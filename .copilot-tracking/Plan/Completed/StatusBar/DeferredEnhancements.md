# Deferred Enhancements

The initial status bar implementation intentionally defers the following items:

- Runtime drag/drop relocation of status bar items between left and right zones.
- Overflow handling for narrow windows (priority collapse, chevrons, or grouped menus).
- Compact mode variants and adaptive typography density.
- Theming variants beyond the default VS Code-inspired presentation.
- User persistence for custom placement/order preferences.

These are intentionally deferred to keep the first release focused on reusable presentation and view-model-driven behavior.
